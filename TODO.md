# Aggiugnere un test nuovo
- metetre i parametri necessari dentro la gerarchia dei test in config.yaml
- se è un dominio nuvo crearlo in src/config/schema/tests_config.py e crere il rispettivo file src/config/schema/domain_X.py
- definire la classe runtime del test in src/core/models/runtime.py
- improtare e settare le config dentro src/engine.py
- creare l'effettivo fiel di test src/tests/domain_X/test_X_X_security_headers_audit.py
